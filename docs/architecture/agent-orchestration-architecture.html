🔴 RAILTRACKS AGENT WORKFLOW NODES — RT_WORKFLOW.PY
1
PERCEIVE
Normalizes raw user input into a structured PlanRequest object. Invokes vision tools to parse images (fridge / meals / receipts) and extract structured inventory and constraint data.
analyze_fridge_vision analyze_meal_vision parse_receipt_items Gemini Vision
2
PRIORITIZE
Identifies ingredients expiring within 1–3 days and extracts hard constraints (allergens, dietary restrictions, budget, time limits). Generates spoilage_alerts for the planner.
ConstraintParser Expiry Detection (<3 days) Hard Constraint Extraction
3
RETRIEVE · RAG Retrieval
Issues a semantic vector search against ChromaDB, prioritizing expiring ingredients. Builds a contextual search query incorporating dietary restrictions and returns the Top-K recipe candidates.
ChromaDB gemini-embedding-001 _build_search_query() retrieve_context()
4
QUERY_RECIPE
Calls the retrieve_recipe_candidates tool to query TheMealDB API. Merges results with RAG candidates, scores and ranks recipes against inventory and constraints, and outputs the final candidate set.
retrieve_recipe_candidates TheMealDB API calculate_meal_macros
5
FORMULATE_PLAN
The Railtracks Agent runs a ReAct loop, calling tools to produce a JSON-structured recommendation. Covers recipe selection, macronutrient breakdown, grocery gap, and cooking workflow decomposition.
Gemini 2.5 Pro (LLM) generate_grocery_gap_tool decompose_cooking_workflow schedule_proactive_prep
6
REFLECT_AND_RETRY
Validates LLM output against all constraints: allergen filtering, vegan/vegetarian keyword checks, budget and time limits. Retries with failure reason injected into the prompt — up to 3 attempts maximum.
reflection.py max_retry = 3 Allergen Filter Budget Check
7
FINALIZE
Assembles AgentPlanOutputV1, calls sync_to_calendar to persist the execution plan, cooking tasks, and prep time windows to the database. Triggers memory layer updates.
sync_to_calendar AgentPlanOutputV1 update_memory_after_recommendations()
↺ REFLECT_AND_RETRY Loop: If output violates a constraint → re-invoke FORMULATE_PLAN with the failure reason appended → retry up to 3 times → gracefully degrade to a partial result if limit is exceeded.

🟨 RAG PIPELINE — RAG_PIPELINE.PY · RT_CONFIG.PY
1
Index Build — _index_recipes()
Recipe title + category + area + ingredients + tags + truncated instructions (4500 chars) concatenated into documents, upserted as Chunk objects into ChromaDB
2
Query Build — _build_search_query()
Prioritize expiring items (<3 days) → expand to full inventory → append dietary restrictions → construct semantic search prompt
3
Vector Search — retrieve_context()
ChromaDB semantic search via Gemini Embedding (gemini-embedding-001), compute distance-based relevance scores, return Top-K candidates
4
LLM Context Injection
Ranked recipe chunks injected as Grounding Context to Gemini 2.5 Pro, augmenting the quality of generated recommendations
✦ Fallback: If vector search fails → invoke keyword_retrieve() → delegate to TheMealDB API keyword matching
